Purpose: help automated agents quickly understand, modify, and verify the FakePlayerPlugin codebase.

Paper 1.21.3+ uses shorthand `MAX_HEALTH`; older servers use `GENERIC_MAX_HEALTH`. Resolved once at class-load via reflection. Use `AttributeCompat.maxHealth()` instead of the raw enum literal wherever `Attribute.MAX_HEALTH` is needed. **`NameTagHelper`** ŌĆö integration shim for the NameTag soft-dep; inner records `BotSkin` and `NickData`; `canRename()` helper; **ClassLoader guard:** never call unless `plugin.isNameTagAvailable()` returns `true` (cached in `onEnable` via `Bukkit.getPluginManager().getPlugin("NameTag") != null`). **`FlagParser`** ŌĆö reusable command arg/flag parser with deprecation aliases, duplicate detection, and conflict detection. Pattern: `new FlagParser(args).deprecate(...).conflicts(...).parse()` ŌĆö use this in new commands instead of ad-hoc arg scanning. `BotRenameHelper` centralises the validate ŌåÆ delete ŌåÆ respawn ŌåÆ restore lifecycle shared by `RenameCommand` and `BotSettingGui`. **`AttributionManager`** ŌĆö lightweight attribution protection; validates original author credit in `plugin.yml` on startup via `AttributionManager.validate(plugin)` (called in `onEnable`). Encoded int-array storage prevents trivial text-search edits. Public API: `getOriginalAuthor()`, `getAttributionMessage()`, `getModrinthLink()`, `getGithubLink()`, `quickAuthorCheck()`, `quickMessageCheck()`, `isIntegrityValid()`, `formatAuthors(List<String>)`. Prefers API-verified data from `AttributionApiManager` when available, falls back to built-in encoded values. **`AttributionApiManager`** ŌĆö async API-based attribution verification; fetches official data from `fpp.wtf/api/attribution` on startup (`init(plugin)` called in `onEnable` after `AttributionManager.validate`). Caches response to `data/attribution-cache.json`; falls back to local cache then built-in encoded data when API unreachable. Public API: `init(Plugin)`, `hasData()`, `isApiReachable()`, `getAuthor()`, `getMessage()`, `getModrinthLink()`, `getGithubLink()`. **`BadwordFilter`** ŌĆö validates and sanitises bot names. Word sources (merged at load time): (1) remote global baseline (`badword-filter.global-list-url`, default CMU list ŌĆö enabled by default), (2) `badword-filter.words` inline list in `config.yml`, (3) `plugins/FakePlayerPlugin/bad-words.yml` (local YAML + custom regex `patterns:` block). Detection passes applied in order: raw (case-insensitive substring), leet (after `0ŌåÆo 1ŌåÆi 3ŌåÆe 4ŌåÆa ŌĆ”` normalisation), aggressive (`auto-detection.enabled: true` ŌĆö duplicate-char collapse), regex (auto-generated repeat/separator patterns + custom patterns). Whitelist: `badword-filter.whitelist` exact names always pass. Auto-rename: `badword-filter.auto-rename: true` substitutes a random clean name instead of hard-blocking. Public API: `BadwordFilter.reload(plugin)`, `isAllowed(name)`, `sanitize(name)`, `findBadword(name)`, `getBadwordCount()`, `getWhitelistCount()`. Called during startup (`BadwordFilter.reload(this)` after `Config.init`, before `Lang.init`), on every `/fpp reload`, and in `SpawnCommand`/`BotRenameHelper` for per-name validation.

Static entry point: `interact(fp, faceLoc, block, plugin, manager, transferFn, onFinally)`. Timing: bot locked + teleported on call; +1 tick container opened via NMS; +3 ticks transfer runs, inventory closed, action lock released, GUI refreshed, `onFinally` called. Note: `BotHeadAI` has been removed; head-AI functionality (player tracking, look-range, turn-speed) is now driven by config keys `head-ai.*` and executed inside `FakePlayerManager`.

- `fakeplayer/FakePlayerManager.java`, `fakeplayer/BotPersistence.java` ŌĆö bot lifecycle, persistence and restore. Key new helpers: `requestNavJump(UUID)` ŌĆö sets a 5-tick jump-hold for navigation (used by `MoveCommand` to prevent the swim-AI from cancelling jumps before physics fires); `clearNavJump(UUID)` ŌĆö clears the pending jump on despawn. **Action lock API (renamed from mining lock):** `lockForAction(UUID, Location)` ŌĆö freezes the bot at the given position and suppresses head-AI for the duration of a block interaction (mining, storage deposit/fetch, right-click use); `unlockAction(UUID)` ŌĆö releases the lock; `isActionLocked(UUID)` ŌĆö query; `updateActionLockRotation(UUID, float yaw, float pitch)` ŌĆö updates the bot's stored yaw/pitch while locked (used by `AttackCommand` mob mode for smooth rotation without releasing the lock). Backed by `actionLockedBots` (`ConcurrentHashMap<UUID, Location>`). Previously named `lockForMining`/`unlockMining`/`isMiningLocked` ŌĆö all old names have been removed. **Navigation lock API (separate from action lock):** `lockForNavigation(UUID)` / `unlockNavigation(UUID)` / `isNavigationLocked(UUID)` ŌĆö backed by `navLockedBots` (`ConcurrentHashSet<UUID>`). Suppresses head-AI rotation while a bot is actively navigating (PathfindingService calls `lockForNavigation` on `navigate()` start and `unlockNavigation` on `cancel()`). Does NOT freeze bot position ŌĆö that is the action lock's job. **`bodyTransitionBots`** (Set<UUID>): `isBodyTransitioning(UUID)` ŌĆö `true` while a bot's NMS body is being swapped (e.g. during world-change re-spawn); prevents duplicate entity registration. **`despawningBotIds`** (`ConcurrentHashMap<UUID, String>`): value is the bot's display name at despawn time; `isDespawning(UUID)` / `getDespawningDisplayName(UUID)` allow event listeners to identify bots being removed and retrieve their display name during the removal window. **`refreshCleanNamePool()`** ŌĆö pre-filters all names from `bot-names.yml` through the badword filter once; the result is cached in `cleanNamePool` and used by `generateName()` instead of running the expensive regex passes inline. Call this after any badword reload (already done automatically on startup and by `ReloadCommand`). **Entity ID index (v1.6.3):** `entityIdIndex` (`ConcurrentHashMap<Integer, FakePlayer>`) ŌĆö secondary O(1) lookup by NMS entity ID, maintained at all add/remove/world-change sites. `getByEntity(Entity entity)` checks this first, then falls back to PDC name tag for world-change recovery (and self-heals the index). Public helpers: `removeFromEntityIndex(int entityId)`, `registerEntityIndex(int entityId, FakePlayer fp)` ŌĆö used by `FakePlayerEntityListener` and `PlayerWorldChangeListener` to keep the index accurate.
- `fakeplayer/BotPathfinder.java` ŌĆö server-side A* grid pathfinder for bot navigation (used by `MoveCommand` and `PathfindingService`). Supports `MoveType` actions: `WALK`, `ASCEND`, `CLIMB`, `DESCEND`, `PARKOUR`, `BREAK`, `PLACE`, `PILLAR`, `SWIM`. Feature flags passed via `PathOptions` record (`parkour`, `breakBlocks`, `placeBlocks`, `avoidWater`, `avoidLava`); defaults are `false`. Key limits are config-driven: `pathfinding.max-range`, `pathfinding.max-nodes`, `pathfinding.max-nodes-extended`. 4) Common tasks & minimal steps (how to implement typical changes)
- Add a new command:
    - Create `src/main/java/me/bill/fakePlayerPlugin/command/MyCommand.java` implementing the command interface used by `CommandManager`.
    - Register it in `FakePlayerPlugin.onEnable()` with `commandManager.register(new MyCommand(...))`.
    - No manual help update is needed ŌĆö `HelpCommand` reads live from `CommandManager`.
    - Ensure `plugin.yml` contains the `fpp` command or sub-command docs (command parsing done by `CommandManager`).
- Add a config option:
    - Add default to `src/main/resources/config.yml`.
    - If needed, update `Config` accessor and `ConfigMigrator` to preserve old configs.
    - New options should be validated in `util/ConfigValidator.java` if they can be misconfigured.
- Add a language message:
    - Add the key and default text to `src/main/resources/language/en.yml`.
    - Access it in code with `Lang.get("your-key")` or `Lang.raw("your-key", "placeholder", "value")`.
    - The file is hot-reloadable ŌĆö `/fpp reload` picks up changes without restart.
- Add a permission node:
    - Declare a new `public static final String` constant in `permission/Perm.java` (e.g. `public static final String MY_FEATURE = "fpp.myfeature";`).
    - Add a full entry under `permissions:` in `plugin.yml` with description and default value (e.g. `op`, `true`, or `false`).
    - Add the node to the `fpp.op` wildcard's `children:` list in `plugin.yml` (so admins inherit it automatically).
    - Use the constant in your command's `getPermission()` and `canUse()` methods: `Perm.has(sender, Perm.MY_FEATURE)`.
    - **Critical:** LuckPerms reads `plugin.yml` at startup for tab-completion ŌĆö permissions missing from `plugin.yml` won't appear in `/lp permission set fpp.<tab>`.
    - Test: restart server, run `/lp user <name> permission set fpp.` and verify your new permission appears in tab-completion.

        - If changing skin behaviour or adding skins: ensure `SkinRepository`/`SkinFetcher` are respected ŌĆö add skin files to `src/main/resources/skins/` for defaults or `plugins/FakePlayerPlugin/skins/` at runtime and call `/fpp reload` to refresh. Implement `getName()`, `getVersion()`, `getDescription()` (default `""`), `getAuthors()` (default empty), `getDependencies()` (default empty), `getSoftDependencies()` (default empty), `getPriority()` (default `100`; lower = earlier load order), `onEnable(FppApi)`, `onDisable()`. No Bukkit `plugin.yml` required ŌĆö the extension loader discovers classes via direct JAR scanning. Extensions are sorted by priority then name before `onEnable()` is called. `ReloadCommand` supports `extensions` target for hot-reload. **Extension config & resource helpers (default methods on `FppExtension`):** `getDataFolder()` returns `extensions/<ExtensionName>/`, `getConfig()` lazy-loads config with JAR defaults, `saveDefaultConfig()` extracts `config.yml` from JAR (tries root then `extension-resources/`), `saveDefaultResources()` extracts all `extension-resources/` files (never overwrites), `saveResource(jarPath)` on-demand single-file extraction, `reloadConfig()` reloads from disk with fresh JAR defaults.
